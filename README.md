Дисциплина «Параллельные Вычислительные Системы» посвящена изучению методов распараллеливания вычислений для многоядерных и многопроцессорных систем. Курс охватывает технологии многопоточного программирования с использованием **OpenMP** и распределённых вычислений с помощью **MPI**.

### Цели изучения
- Понимание архитектуры параллельных систем
- Освоение технологий распараллеливания (OpenMP, MPI)
- Получение навыков анализа и оптимизации производительности

**Дата выполнения:** 2024  
**Файл:** `pr0/pr0.cpp`

#### Цель
Освоение методов замера времени выполнения кода для последующего сравнения производительности.

#### Задачи
- Изучение `std::chrono` для точных замеров
- Создание эталонной последовательной версии
- Расчёт ускорения (speedup) и эффективности

#### Код замера времени
```cpp
#include <chrono>
#include <iostream>

auto start = std::chrono::high_resolution_clock::now();

// ... выполняемый код ...

auto end = std::chrono::high_resolution_clock::now();
auto duration = std::chrono::duration_cast<std::chrono::milliseconds>(end - start);

std::cout << "Time: " << duration.count() << " ms" << std::endl;
```

#### Метрики производительности
- **Speedup (Ускорение):** `S = T_seq / T_par`
- **Efficiency (Эффективность):** `E = S / p`, где p — количество процессоров

### Практическая работа 1: Распараллеливание циклов с OpenMP

**Дата выполнения:** 2024  
**Файл:** `pr1/pr1.cpp`

#### Цель
Изучение базовых директив OpenMP для распараллеливания циклов.

#### Задачи
- Распараллеливание цикла for
- Настройка количества потоков
- Распределение итераций (schedule)

#### Ключевые директивы
```cpp
#include <omp.h>

// Установка количества потоков
omp_set_num_threads(4);

// Распараллеливание цикла
#pragma omp parallel for
for (int i = 0; i < n; i++) {
    c[i] = a[i] + b[i];
}

// С указанием количества потоков
#pragma omp parallel for num_threads(8)
for (int i = 0; i < n; i++) {
    // ...
}
```

#### Планирование (Scheduling)
```cpp
// Статическое распределение (по умолчанию)
#pragma omp parallel for schedule(static)

// Динамическое распределение (для неравномерной нагрузки)
#pragma omp parallel for schedule(dynamic, chunk_size)

// Руководство (программист решает)
#pragma omp parallel for schedule(guided)
```

#### Результат
- Ускорение в 2-4 раза на 4-ядерном процессоре
- Понимание накладных расходов на создание потоков

### Практическая работа 2: Критические секции и редукция

**Дата выполнения:** 2024  
**Файл:** `pr2/pr2.cpp`

#### Цель
Изучение механизмов синхронизации для защиты общих данных.

#### Задачи
- Защита критических секций (`#pragma omp critical`)
- Использование редукции (`reduction` clause)
- Атомарные операции (`#pragma omp atomic`)

#### Пример: критическая секция
```cpp
int sum = 0;

#pragma omp parallel for
for (int i = 0; i < n; i++) {
    #pragma omp critical
    {
        sum += data[i];  // Только один поток выполняет одновременно
    }
}
```

#### Пример: редукция (эффективнее)
```cpp
int sum = 0;

#pragma omp parallel for reduction(+:sum)
for (int i = 0; i < n; i++) {
    sum += data[i];  // Каждый поток имеет локальную копию
}
```

#### Операции редукции
| Операция | Синтаксис | Начальное значение |
|---|---|---|
| Сложение | `reduction(+:var)` | 0 |
| Умножение | `reduction(*:var)` | 1 |
| Максимум | `reduction(max:var)` | -∞ |
| Минимум | `reduction(min:var)` | +∞ |
| AND | `reduction(&:var)` | Все 1 |
| OR | `reduction(|:var)` | Все 0 |

#### Результат
- Понимание проблем состояния гонки (race condition)
- Умение выбирать между critical и reduction

### Практическая работа 3: MPI — Отправка и приём сообщений

**Дата выполнения:** 2024  
**Файл:** `pr3/pr3.cpp`

#### Цель
Освоение базовых функций MPI для межпроцессного взаимодействия.

#### Задачи
- Инициализация MPI среды
- Определение ранга процесса и размера коммуникатора
- Точечная передача сообщений (Send/Recv)

#### Структура MPI программы
```cpp
#include <mpi.h>

int main(int argc, char** argv) {
    MPI_Init(&argc, &argv);
    
    int rank, size;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);  // Номер процесса
    MPI_Comm_size(MPI_COMM_WORLD, &size);  // Количество процессов
    
    if (rank == 0) {
        // Процесс 0 отправляет данные
        int data = 42;
        MPI_Send(&data, 1, MPI_INT, 1, 0, MPI_COMM_WORLD);
    } else if (rank == 1) {
        // Процесс 1 получает данные
        int data;
        MPI_Recv(&data, 1, MPI_INT, 0, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
        std::cout << "Received: " << data << std::endl;
    }
    
    MPI_Finalize();
    return 0;
}
```

#### Параметры MPI_Send / MPI_Recv
```cpp
MPI_Send(
    void* data,          // Адрес буфера
    int count,           // Количество элементов
    MPI_Datatype type,   // Тип данных (MPI_INT, MPI_DOUBLE)
    int dest,            // Ранг получателя
    int tag,             // Тег сообщения
    MPI_Comm comm        // Коммуникатор
);

MPI_Recv(
    void* data,
    int count,
    MPI_Datatype type,
    int source,          // Ранг отправителя (MPI_ANY_SOURCE)
    int tag,             // Тег (MPI_ANY_TAG)
    MPI_Comm comm,
    MPI_Status* status   // Информация о сообщении
);
```

#### Результат
- Понимание распределённой памяти (в отличие от общей в OpenMP)
- Навыки отладки многопроцессных программ

### Практическая работа 4: MPI — Коллективные операции

**Дата выполнения:** 2024  
**Файл:** `pr4/pr4.cpp`

#### Цель
Изучение коллективных операций MPI для эффективной работы с группами процессов.

#### Задачи
- Вещание (Broadcast)
- Сбор данных (Gather)
- Рассылка (Scatter)
- Редукция (Reduce)

#### Broadcast — рассылка от одного всем
```cpp
int data;
if (rank == 0) {
    data = 100;  // Процесс 0 устанавливает значение
}

MPI_Bcast(&data, 1, MPI_INT, 0, MPI_COMM_WORLD);
// Теперь все процессы имеют data = 100
```

#### Gather — сбор данных к одному процессу
```cpp
int local_value = rank * 10;
int* all_values = NULL;

if (rank == 0) {
    all_values = new int[size];
}

MPI_Gather(&local_value, 1, MPI_INT, all_values, 1, MPI_INT, 0, MPI_COMM_WORLD);

// Процесс 0 теперь имеет массив всех значений
```

#### Scatter — рассылка разных данных
```cpp
int* send_data = NULL;
int recv_data;

if (rank == 0) {
    send_data = new int[size];
    for (int i = 0; i < size; i++) {
        send_data[i] = i * 10;
    }
}

MPI_Scatter(send_data, 1, MPI_INT, &recv_data, 1, MPI_INT, 0, MPI_COMM_WORLD);
// Каждый процесс получает свой элемент
```

#### Reduce — редукция данных
```cpp
int local_sum = compute_local_sum();
int global_sum;

MPI_Reduce(&local_sum, &global_sum, 1, MPI_INT, MPI_SUM, 0, MPI_COMM_WORLD);

// Процесс 0 имеет глобальную сумму
```

### Практическая работа 5: MPI — Распределённые вычисления

**Дата выполнения:** 2024  
**Файл:** `pr5/pr5.cpp`

#### Цель
Применение MPI для решения вычислительной задачи на нескольких процессах.

#### Задача: Параллельное суммирование массива
```cpp
const int N = 1000000;
int* data = new int[N];
// ... инициализация данных ...

int local_size = N / size;
int local_start = rank * local_size;
int local_sum = 0;

// Локальное вычисление
for (int i = local_start; i < local_start + local_size; i++) {
    local_sum += data[i];
}

// Глобальная редукция
int global_sum;
MPI_Reduce(&local_sum, &global_sum, 1, MPI_INT, MPI_SUM, 0, MPI_COMM_WORLD);

if (rank == 0) {
    std::cout << "Total sum: " << global_sum << std::endl;
}
```

#### Результат
- Ускорение пропорционально количеству процессов
- Понимание накладных расходов на коммуникацию

## 🎯 Приобретённые навыки

### OpenMP
- **Директивы:** `parallel for`, `critical`, `atomic`, `reduction`
- **Планирование:** static, dynamic, guided
- **Синхронизация:** барьеры, критические секции
- **Переменные:** shared, private, firstprivate, lastprivate

### MPI
- **Инициализация:** `MPI_Init`, `MPI_Finalize`
- **Точечная связь:** `MPI_Send`, `MPI_Recv`
- **Коллективные операции:** `MPI_Bcast`, `MPI_Gather`, `MPI_Scatter`, `MPI_Reduce`
- **Типы данных:** `MPI_INT`, `MPI_DOUBLE`, `MPI_FLOAT`

### Анализ производительности
- **Speedup:** S = T_seq / T_par
- **Efficiency:** E = S / p
- **Amdahl's Law:** ограничение ускорения последовательной частью

## 📊 Сравнение OpenMP и MPI

| Характеристика | OpenMP | MPI |
|---|---|---|
| **Модель памяти** | Общая (Shared Memory) | Распределённая (Distributed) |
| **Уровень** | Потоки (threads) | Процессы (processes) |
| **Масштабируемость** | Один узел (до ~64 ядер) | Кластер (тысячи узлов) |
| **Программирование** | Директивы (#pragma) | Функции (MPI_Send/Recv) |
| **Сложность** | Низкая | Средняя/Высокая |
| **Применение** | Многоядерные CPU | Кластеры, суперкомпьютеры |
